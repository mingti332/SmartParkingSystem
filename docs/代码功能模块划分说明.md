# 智能停车位共享与预约管理系统 - 代码功能模块划分说明

## 1. 分层结构总览

当前项目采用典型的 `UI -> Service -> DAO -> DB` 分层。

- UI 层：`com.parking.app`
  - `MainApp`：登录/注册入口与主界面切换
  - `DashboardFactory`：管理员/车位所有者/车主功能页构建与事件绑定
- 业务层：`com.parking.service` + `com.parking.service.impl`
  - 负责参数校验、业务规则、事务编排、异常语义化
- 数据访问层：`com.parking.dao` + `com.parking.dao.impl`
  - 负责 SQL、分页、排序、多表查询与更新
- 实体层：`com.parking.entity`
  - 对应数据库表实体
- 配置层：`com.parking.config.DbUtil`
  - JDBC 连接管理

## 2. 功能模块划分（按业务域）

### 模块A：用户与权限管理

- 目标
  - 登录、注册、用户查询、启用/禁用、重置密码、删除用户（含保护规则）
- UI入口
  - `DashboardFactory.userTab()`（管理员）
  - `MainApp.showLogin()`（登录/注册）
- 业务接口
  - `AuthService`
  - `UserAdminService`
- DAO
  - `UserDao`
- 关联表
  - `Users`
- 核心规则
  - 管理员ID（1）不可删除
  - 删除前校验：是否存在、是否有车位、是否在场、是否有关联记录
  - 手机号支持特定查询策略（完整11位或后4位）

### 模块B：停车场管理

- 目标
  - 停车场新增/修改/删除/分页查询
- UI入口
  - `DashboardFactory.parkingLotTab()`
- 业务接口
  - `ParkingLotService`
- DAO
  - `ParkingLotDao`
- 关联表
  - `ParkingLots`
- 核心规则
  - 关键字段校验：名称、地址、总车位、营业时段

### 模块C：车位管理与共享时段

- 目标
  - 车位新增/修改/删除/查询
  - 车位所有者维护共享时段
  - 按状态/类型/停车场/关键字筛选
- UI入口
  - `DashboardFactory.parkingSpaceTab()`（管理员）
  - `DashboardFactory.ownerView()`（所有者）
- 业务接口
  - `ParkingSpaceService`
- DAO
  - `ParkingSpaceDao`
- 关联表
  - `ParkingSpaces`
- 核心规则
  - 共享时段合法性校验
  - 支持“可预约时段可用车位”查询

### 模块D：预约管理（核心）

- 目标
  - 查询可用车位、提交预约、取消预约、查看我的预约、查看名下预约
- UI入口
  - `DashboardFactory.reservationTab()`（车主）
  - `DashboardFactory.ownerView()`（所有者）
- 业务接口
  - `ReservationService`
- DAO
  - `ReservationDao`
- 关联表
  - `Reservations`
- 核心规则
  - 时间段冲突检测
  - 预约时间格式与有效范围校验
  - 取消预约状态约束

### 模块E：停车入场/出场与计时

- 目标
  - 入场登记、出场结算触发、停车记录查询
- UI入口
  - `DashboardFactory.parkingTab()`（车主）
- 业务接口
  - `ParkingRecordService`
- DAO
  - `ParkingRecordDao`
- 关联表
  - `ParkingRecords`
- 核心规则
  - 入场允许“有预约/无预约”场景
  - 出场后自动计算时长与费用
  - 在场车辆通过 `exit_time IS NULL` 判断

### 模块F：计费规则管理

- 目标
  - 计费规则增删改查（按时/按次）
- UI入口
  - `DashboardFactory.pricingRuleTab()`
- 业务接口
  - `PricingRuleService`
- DAO
  - `PricingRuleDao`
- 关联表
  - `PricingRules`
- 策略模式实现
  - `HourlyPricingStrategy`
  - `FixedPricingStrategy`
- 核心规则
  - 按时需单价+单位时长
  - 按次需固定价
  - 车位类型匹配规则

### 模块G：支付记录

- 目标
  - 出场支付后生成支付记录，车主可查询支付明细
- UI入口
  - `DashboardFactory.parkingTab()` 的“我的支付记录”
- 业务接口
  - `PaymentService`
- DAO
  - `PaymentRecordDao`
- 关联表
  - `PaymentRecords`
- 核心规则
  - 支付状态显示本地化

### 模块H：收益管理与结算审核

- 目标
  - 所有者查看收益明细与总收益
  - 管理员进行结算审核
- UI入口
  - `DashboardFactory.ownerView()`（收益明细）
  - `DashboardFactory.settlementTab()`（结算审核）
- 业务接口
  - `RevenueService`
- DAO
  - `RevenueRecordDao`
- 关联表
  - `RevenueRecords`
- 核心规则
  - 总收益汇总显示（允许负值）
  - 待结算 -> 已结算 状态流转

### 模块I：统计报表

- 目标
  - 收入排行、预约排行、高峰时段
- UI入口
  - `DashboardFactory.reportTab()`
- 业务接口
  - `ReportService`
- DAO
  - `ReportDao`
- 关联表
  - `ParkingLots/ParkingSpaces/Reservations/ParkingRecords/RevenueRecords`
- 核心规则
  - 高峰时段显示为 `HH:00-HH:59`（非纯数字）

## 3. 关键业务链路（代码调用视角）

### 链路1：预约

`reservationTab -> ReservationService.reserve -> ReservationDao`  
同时会联动车位状态、冲突检测与时间合法性校验。

### 链路2：停车与支付

`parkingTab.entry -> ParkingRecordService.entry -> ParkingRecordDao`  
`parkingTab.exitAndPay -> ParkingRecordService.exitAndPay -> PricingRule/Payment/Revenue DAO`

### 链路3：收益结算

`settlementTab -> RevenueService.settleRevenue -> RevenueRecordDao.settleById`

### 链路4：管理员用户维护

`userTab -> UserAdminService -> UserDao`  
含分页、角色过滤、状态过滤、删除保护。

## 5. 可继续优化的模块拆分建议（不影响当前功能）

当前功能已经按业务实现完整，但 `DashboardFactory` 仍较大。建议下一步拆分为：

- `com.parking.app.admin.AdminUserView`
- `com.parking.app.admin.AdminLotView`
- `com.parking.app.admin.AdminSpaceView`
- `com.parking.app.admin.AdminPricingView`
- `com.parking.app.admin.AdminSettlementView`
- `com.parking.app.admin.AdminReportView`
- `com.parking.app.owner.OwnerView`
- `com.parking.app.carowner.ReservationView`
- `com.parking.app.carowner.ParkingView`

这样每个模块一个类，便于维护、答辩讲解和后续测试。
